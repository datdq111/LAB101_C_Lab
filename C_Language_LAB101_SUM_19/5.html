Memcached. C.L.P0010
Approved Long Assignment
C Basic Manual Grading Memcached.

Memcached is a high-performance, distributed memory object caching system, generic in nature, but originally intended for use in speeding up dynamic web applications by alleviating database load.

Memcached is an in-memory key-value store for small chunks of arbitrary data (strings, objects) from results of database calls, API calls, or page rendering.

There are some main functions that we use with Memcache and they are as follows:

  • Store data: sets a given key with a given value
  • Retrieve item: gets the value for a specified key
  • Delete item: remove a key and cache data
  • Flush all existing items: removes all keys and cached data
  • Exit the program.

 

Function details:

  • Function 1: When we store data using Memcache there are 3 parts to storing the data, those three parts are the key, the value, and the expiry time of the cached item. The key is used to reference the stored data.
    • The key that will be associated with the item.
    • The variable to store. Strings and integers are stored as is, other types are stored serialized.
    • Expiration time of the item. If it's equal to zero, the item will never expire. You can also use Unix timestamp or a number of seconds starting from current time, but in the latter case the number of seconds may not exceed 2592000 (30 days).
  • Function 2: Retrieve item, returns previously stored data if an item with such key exists.
  • Function 3: Deletes an item with the key.
  • Flush all existing items: The flush will expire all items up to the ones set within the same second.
